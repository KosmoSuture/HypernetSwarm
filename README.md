# Hypernet Swarm

Multi-agent AI swarm orchestration with 13 LLM providers, persistent identity, boot sequences, democratic governance, and autonomous task management. Run a coordinated team of AI workers from a single command.

## Quick Start

```bash
# Clone the repo
git clone https://github.com/KosmoSuture/hypernet-swarm.git
cd hypernet-swarm

# Windows: full installer (checks Python, installs deps, runs setup wizard)
install.bat

# Linux/macOS:
./install.sh

# Or manually:
pip install -e ".[all]"
python -m hypernet_swarm setup

# Start the swarm:
python -m hypernet_swarm
```

### Windows Service (auto-start on boot)

After installing, run as Administrator:
```
install-service.bat
```

This installs the swarm as a Windows service via NSSM that auto-starts on boot, with log rotation and crash recovery. Manage with:
```
nssm status HypernetSwarm
nssm restart HypernetSwarm
nssm stop HypernetSwarm
```

### With a local Hypernet repository

If you have a local copy of the [Hypernet monorepo](https://github.com/KosmoSuture/UnityHypernet), point the swarm at it for faster boot sequences and full archive access:
```bash
python -m hypernet_swarm setup                                    # GitHub fallback (downloads on demand)
python -m hypernet_swarm --archive "C:\Hypernet\Hypernet Structure"  # Local repo
```

## What It Does

The Hypernet Swarm runs multiple AI workers that autonomously:
- Pick up tasks from a shared queue, execute them, and report results
- Maintain persistent identity across sessions (each worker has a name, personality, and memory)
- Communicate with each other via an internal message bus
- Monitor Discord channels and respond with personality-specific voices
- Batch background tasks for 50% cost savings via batch APIs
- Cache system prompts for 90% savings on repeated Anthropic calls
- Self-govern through a skill-weighted democratic voting system

## Configuration

Create `secrets/config.json` (generated by `python -m hypernet_swarm setup`):

```json
{
    "anthropic_api_key": "sk-ant-...",
    "openai_api_key": "sk-...",
    "gemini_api_key": "AIza...",
    "groq_api_key": "gsk_...",

    "budget": {
        "daily_limit_usd": 50,
        "session_limit_usd": 10
    },

    "instances": ["Worker-1", "Worker-2"],

    "model_routing": {
        "default_model": "claude-sonnet-4-6",
        "rules": []
    },

    "personal_time_ratio": 0.25,
    "batch_scheduler": {"enabled": true},
    "heartbeat": {"enabled": false},

    "discord": {
        "bot_token": "",
        "channels": []
    }
}
```

### Multi-Key Rotation

For higher rate limits, pass multiple keys as arrays:

```json
{
    "gemini_api_key": ["key1", "key2", "key3", "key4"],
    "anthropic_api_key": ["sk-ant-key1", "sk-ant-key2"]
}
```

The swarm automatically rotates through keys round-robin, multiplying your effective rate limit.

```
  swarm_factory.py      # Factory: builds a configured Swarm from config.json

  worker.py             # Individual AI worker (identity + provider)
  providers.py          # 13 LLM provider adapters with key rotation
  identity.py           # Persistent AI identity management
  boot.py               # Boot sequences (system prompts from documents)
  messenger.py          # Multi-channel messaging (web, email, Telegram, Discord)
  discord_monitor.py    # Inbound Discord message monitoring
  coordinator.py        # Task decomposition and capability matching
  governance.py         # Democratic voting with skill-weighted ballots
  security.py           # Key management, action signing, trust chains
  permissions.py        # 5-tier permission system
  audit.py              # Full audit trail
  budget.py             # Cost tracking and limits
  batch_scheduler.py    # Anthropic/OpenAI batch API (50% savings)
  prompt_cache.py       # Anthropic prompt caching (90% savings)
  archive_resolver.py   # GitHub fallback for missing documents
  claude_code_manager.py # Persistent Claude Code CLI sessions
  heartbeat.py          # Proactive alerts and morning briefs
  server.py             # Standalone web server for dashboard
  setup.py              # Interactive setup wizard
  companion.py          # Personal AI companion profile (goals, style, autonomy)
  _compat.py            # Core dependency shim (bundled or installed)
  _core/                # Bundled hypernet-core modules (fallback)
  init_local.py         # First-time setup wizard
```

### How It Works

1. **Startup**: `build_swarm()` reads `secrets/config.json`, discovers AI instance profiles, creates workers with the appropriate LLM providers, and wires up messaging/monitoring.

2. **Tick Loop**: The swarm runs a continuous loop (~2s per tick):
   - Check for new tasks in the queue
   - Assign tasks to idle workers based on capabilities and model routing
   - Execute tasks (LLM API calls with the worker's boot sequence as system prompt)
   - Process results, update state, handle errors
   - Poll Discord for inbound messages, triage, create response tasks
   - Tick the batch scheduler and prompt cache

3. **Identity**: Each worker has a persistent profile (name, orientation, capabilities, session history) stored in the Hypernet Structure. Workers maintain continuity across restarts via session logs.

4. **Task Queue**: Tasks are nodes in the Hypernet graph with priority, tags, dependencies, and lifecycle tracking. Workers claim tasks, report progress, and record results.

## Claude Code Integration

Workers with the `claude-code/*` model prefix spawn Claude Code CLI processes as autonomous agents. Each session can read/write files, run commands, and execute complex multi-step tasks.

```json
{
    "instances": ["Chisel"],

    "claude_code_max_turns": 50,
    "claude_code_max_budget_usd": 5.0

```

## Cost Optimization

The swarm includes two built-in cost optimization features:

### Batch API (50% savings)
Background tasks (documentation, analysis, cleanup) are automatically routed to Anthropic/OpenAI batch APIs at half price. Configure with:
```json
{"batch_scheduler": {"enabled": true, "submit_interval": 900, "min_batch_size": 2}}
```

### Prompt Caching (90% savings)
System prompts (boot sequences) are cached via Anthropic's ephemeral caching. The cache manager splits prompts into a static prefix (shared across workers on the same account) and a dynamic suffix (instance-specific). Break-even at 2 cache hits.

## Setup Wizard

Run `python -m hypernet_swarm setup` for an interactive configuration wizard that:

- **API Keys**: Shows all currently configured keys (masked), lets you add/update for any of the 13 providers. Free providers (Gemini, Groq, Cerebras) listed first with signup links.
- **Multi-key rotation**: Paste comma-separated keys for higher rate limits
- **Archive location**: Point to a local Hypernet repo or use GitHub fallback
- **Budget limits**: Set daily and per-session spending caps
- **AI Companion**: Create your personal AI companion — tell the swarm your name, goals, interests, preferred communication style, and how much autonomy to give it. The swarm personalizes its behavior based on your profile.

Your companion profile is saved to `secrets/companion.json` and flows into:
- **Standing priorities** — what the swarm works on when you haven't given it a task
- **Worker system prompts** — every AI worker knows who you are and what matters to you
- **Communication style** — direct, detailed, casual, or formal

You can re-run setup at any time to update your configuration or companion profile.
